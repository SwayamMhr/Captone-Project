<!-- Project Title -->
<h1 align="center">Two Wheeler Rental Management System</h1>

<!-- Introduction Section -->
<p>
  - 🔭 I worked as the <b>Backend Developer</b> using <b>Node.js</b> and <b>Express.js</b> for this Capstone project[cite: 1].
</p>

<p>
  This project is a high-performance MERN-stack backend designed to streamline two-wheeler rentals through a secure API and robust data management[cite: 1].
</p>

<hr />

<!-- Backend Tech Stack -->
<h3>🛠️ Backend Tech Stack</h3>
<ul>
  <li><b>Runtime:</b> Node.js[cite: 1]</li>
  <li><b>Framework:</b> Express.js[cite: 1]</li>
  <li><b>Database:</b> MongoDB with Mongoose[cite: 1]</li>
  <li><b>Security:</b> JWT (Authentication) & Bcrypt (Password Hashing)[cite: 1]</li>
  <li><b>Validation:</b> Yup[cite: 1]</li>
</ul>

<hr />

<!-- Key Backend Modules -->
<h3>🗝️ Key Backend Modules</h3>

<h4>1. User Management</h4>
<ul>
  <li><b>Secure Auth:</b> Facilitates registration and login with fully encrypted passwords[cite: 1].</li>
  <li><b>JWT Integration:</b> Generates 7-day access tokens to maintain secure user sessions[cite: 1].</li>
  <li><b>Strict Validation:</b> Ensures unique email registration and enforces password complexity[cite: 1].</li>
</ul>

<h4>2. KYC (Know Your Customer) System</h4>
<ul>
  <li><b>Identity Verification:</b> A dedicated endpoint for users to submit required rental documentation[cite: 1].</li>
  <li><b>Auth-Guarded:</b> Protected by middleware to ensure only verified, logged-in users can submit data[cite: 1].</li>
</ul>

<h4>3. Vehicle & Rental Management</h4>
<ul>
  <li><b>CRUD Operations:</b> Comprehensive logic for managing the vehicle lifecycle (Add, Edit, Delete)[cite: 1].</li>
  <li><b>Advanced Querying:</b> Implements MongoDB aggregate pipelines for high-speed, paginated vehicle searches[cite: 1].</li>
  <li><b>System Safety:</b> Custom middleware validates MongoDB IDs and user permissions before any data modification[cite: 1].</li>
</ul>

<hr />

<!-- File Structure -->
<h3>📂 File Structure Highlights</h3>
<ul>
  <li><code>index.js</code>: Primary server entry point and database connection logic[cite: 1].</li>
  <li><code>/user</code>: Account-related controllers and database models[cite: 1].</li>
  <li><code>/product</code>: Core logic for inventory management and vehicle availability[cite: 1].</li>
  <li><code>/middleware</code>: Security layers including authentication guards and ID validators[cite: 1].</li>
</ul>

<hr />

<!-- Quick Start -->
<h3>🚀 Quick Start</h3>
<ol>
  <li>Navigate to the directory: <code>cd backend</code>[cite: 1]</li>
  <li>Install all dependencies: <code>npm install</code>[cite: 1]</li>
  <li>Configure environment: Setup <code>.env</code> with your <code>MONGO_URL</code> and <code>SECRET_KEY</code>[cite: 1]</li>
  <li>Launch the server: <code>npm run dev</code>[cite: 1]</li>
</ol>
